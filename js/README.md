# 📓 Meu Caderno JavaScript

Este é o meu caderno pessoal de aprendizado de JavaScript.
Tudo aqui foi escrito por mim, com orientação do mentor.
É o registro vivo da minha evolução como desenvolvedor.

---

## 📁 Como está organizado

```
js/
  lessons/          ← arquivos de lição criados pelo mentor antes de cada exercício
    hof/            ← Higher Order Functions
    map/
    filter/
    reduce/
    ...

  exercises/        ← meus exercícios escritos à mão
    array_methods_01.js
    ...

  projects/         ← projetos autorais que eu idealizei e construí
  
  utils/            ← funções reutilizáveis que fui criando ao longo do tempo
```

---

## 📖 Como ler este caderno

### As lições (`lessons/`)
Cada conceito tem sua própria pasta com até 4 arquivos em progressão:

| Arquivo | O que é |
|---|---|
| `01-o-problema.md` | O problema que existia antes do conceito. Leia primeiro. |
| `02-a-solucao.md` | O conceito novo. Comparação antes vs depois. |
| `03-como-usar.md` | Sintaxe, variações, erros comuns e link para pesquisa. |
| `04-exercicios-progressivos.js` | Três exercícios em nível crescente. |

**Regra:** nunca pule direto para o exercício sem ler os três `.md` antes.

### Os exercícios (`exercises/`)
São os arquivos `.js` onde eu escrevo meu código.
Quando peço revisão ao mentor, ele escreve comentários educacionais
diretamente dentro desses arquivos no formato:

```javascript
/*
O QUE ESTÁ ACONTECENDO:
...

DICA:
...

EXEMPLO SIMPLIFICADO:
...

O QUE VOCÊ DEVE FAZER:
...
*/
```

Esses comentários ficam no arquivo para sempre — são parte do aprendizado.

---

## 🔁 Como estudar com este caderno

O fluxo correto para cada conceito novo é sempre este:

```
1. Peço ao mentor: "Crie os arquivos de lição para [conceito]"
2. Leio os três .md com calma, sem pressa
3. Faço minhas anotações com minhas próprias palavras
4. Peço avaliação: "Avalie minhas notas"
5. Recebo nota e liberação para o exercício
6. Escrevo meu código no arquivo .js
7. Peço revisão: "Revise meu código"
8. Leio os comentários e corrijo
9. Commito: git add . && git commit -m "feat: ..." && git push
10. Peço: "Atualize meu histórico"
```

---

## 📝 Como fazer minhas anotações

Antes de pedir avaliação, escrevo minhas notas respondendo:

- **O que este conceito faz?** (em uma frase)
- **Por que ele foi inventado?** (qual problema resolve)
- **Quando usar?** (e quando NÃO usar)
- **Como se diferencia de conceitos parecidos?**
- **Minha analogia do mundo real**

O mentor avalia essas respostas com nota de 0 a 10.
Só sigo para o exercício com nota ≥ 7.
